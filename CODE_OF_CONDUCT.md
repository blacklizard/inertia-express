# Code of Conduct

## The short version

Be decent. Focus on the work.

## Expected behaviour

- Give and receive feedback on technical merit, not personality.
- Assume good faith; ask before concluding bad intent.
- Keep discussion scoped to the issue or PR at hand.
- Credit others' contributions accurately.

## Not acceptable

- Harassment, personal attacks, or derogatory language — towards anyone, for any reason.
- Dismissing contributors based on identity rather than the quality of their work.
- Spam, off-topic self-promotion, or deliberate disruption.

## Scope

This applies across all project spaces: GitHub issues, pull requests, discussions, and any other venue where the project is represented.

## Enforcement

Violations can be reported to **jeevanlx@gmail.com**. Reports are handled confidentially. Depending on severity, the response ranges from a private note to a permanent ban from project spaces.
