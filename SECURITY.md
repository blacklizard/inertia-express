# Security Policy

## Supported versions

Only the latest release of `@blacklizard/inertia-express` receives security fixes. Patch releases are published promptly for confirmed vulnerabilities.

## Reporting a vulnerability

**Do not open a public GitHub issue for security reports.**

Email **jeevanlx@gmail.com** with:

- A description of the vulnerability and its potential impact.
- Steps to reproduce or a minimal proof-of-concept.
- Affected versions, if known.

You'll receive an acknowledgement within **72 hours**. If the report is confirmed, a fix will be prepared and a patched release published before any public disclosure. Credit is given in the release notes unless you prefer to remain anonymous.

## Scope

This policy covers the packages published under `@blacklizard/inertia-express`, `@blacklizard/inertia-ssr-worker`, and `@blacklizard/inertia-cache-redis`. Vulnerabilities in upstream dependencies (Express, Inertia.js, Vite, etc.) should be reported to their respective maintainers.
